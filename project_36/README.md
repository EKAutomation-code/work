
# Классификация цифр 0 и 6 из MNIST с использованием PCA и стекинга

Проект по бинарной классификации изображений рукописных цифр 0 и 6 из датасета MNIST. Используется метод главных компонент (PCA) для снижения размерности и стекинг (Stacking) из трёх моделей машинного обучения.

## Структура проекта

```
project_36/
├── project_36.py # Скрипт с моделью
├── MNIST_train.csv.gz # Обучающая выборка (загрузить отдельно)
├── MNIST_test.csv.gz # Тестовая выборка (загрузить отдельно)
└── README.md # Описание проекта
```

## Что делает скрипт

1. Загружает обучающую и тестовую выборки MNIST
2. Фильтрует только цифры **0** и **6** (бинарная классификация)
3. Применяет **PCA (Principal Component Analysis)** для снижения размерности с 784 признаков до M компонент
4. Обучает **стекинг-модель** из:
   - Random Forest
   - SVC (калиброванный)
   - Логистическая регрессия (финальный классификатор)
5. Выводит метрики качества: точность, полноту, F1-меру
6. Показывает матрицу ошибок (confusion matrix)

## Технологии

- Python 3.x
- `pandas` — работа с данными
- `numpy` — математические операции
- `scikit-learn`:
  - `PCA` — снижение размерности
  - `StackingClassifier` — ансамблевое обучение
  - `RandomForestClassifier` — базовый классификатор
  - `SVC` — метод опорных векторов
  - `LogisticRegression` — финальный классификатор
  - `CalibratedClassifierCV` — калибровка вероятностей SVC

## Настройка

Чтобы изменить порог PCA, отредактируйте строку в project_36.py:

### Было: 87%
```python
M = np.where(cumsum > 0.87)[0][0] + 1
```

### Стало: 95%
```python
M = np.where(cumsum > 0.95)[0][0] + 1
```

Чтобы изменить random_state, отредактируйте строку в project_36.py:

### Было: 19
```python
random_state=19
```

### Стало: 42
```python
random_state=42
```

Чтобы изменить кол-во фолдов в кросс-валидации, отредактируйте строку в project_36.py:

### Было: 5
```python
cv=5
```

### Стало: 10
```python
cv=10
```
